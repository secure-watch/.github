# 보안 정책

[English](SECURITY.md)

## 지원 상태

Secure Watch는 초기 개발 단계입니다. 첫 공개 릴리스 전에 지원 버전과 지원 범위를
명시합니다.

## 취약점 신고

보안 취약점이나 민감한 정보는 공개 Issue나 Discussion에 게시하지 않습니다.

해당 저장소의 **Security** 탭에서 비공개 취약점 신고를 제출합니다. 비공개 신고를
사용할 수 없다면 공개 Issue에서 maintainer에게 비공개 신고 활성화 또는 선호하는
비공개 연락 경로를 요청합니다. 해당 Issue에는 취약점 세부 정보를 포함하지
않습니다.

가능한 범위에서 다음 내용을 포함합니다.

- 영향을 받는 저장소와 버전 또는 commit
- 재현 조건과 영향 범위
- 최소 재현 절차 또는 proof of concept
- 알려진 완화 방법
