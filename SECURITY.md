# Security Policy

[한국어 안내](#한국어)

## Support status

Secure Watch is in early development. The supported versions and support scope will be documented before the first public release.

## Reporting a vulnerability

Do not post security vulnerabilities or sensitive information in public Issues or Discussions.

Submit a private vulnerability report through the repository's **Security** tab. If private reporting is unavailable, contact a maintainer without sharing public details.

Where possible, include:

- the affected repository and version or commit
- reproduction conditions and impact
- a minimal reproduction or proof of concept
- known mitigations

## 한국어

### 지원 상태

Secure Watch는 초기 개발 단계입니다. 첫 공개 릴리스 전에 지원 버전과 지원 범위를 명시합니다.

### 취약점 신고

보안 취약점이나 민감한 정보는 공개 Issue나 Discussion에 게시하지 않습니다.

해당 저장소의 **Security** 탭에서 비공개 취약점 신고를 제출합니다. 비공개 신고를 사용할 수 없다면 공개 세부 정보 없이 maintainer에게 연락합니다.

가능한 범위에서 다음 내용을 포함합니다.

- 영향을 받는 저장소와 버전 또는 commit
- 재현 조건과 영향 범위
- 최소 재현 절차 또는 proof of concept
- 알려진 완화 방법
